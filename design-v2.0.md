# 部门任务管理看板系统 V2.0 — 设计书

在 V1.0 基础上新增甘特图视图，支持任务分类分组、自定义优先级、可拖拽时间轴交互。

---

## 一、数据模型变更

### tasks（任务）新增字段
| 字段 | 类型 | 说明 |
|------|------|------|
| startDate | string(ISO) | 任务开始日期 |
| endDate | string(ISO) | 任务结束日期 |
| category | string | 任务分类ID |

> `deadline` 字段保留，语义更名为「期望时间」（字段名不变，UI 显示为"期望时间"）

### categories（任务分类）— 新增集合
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string(uuid) | 主键 |
| name | string | 分类名称（如：开发、测试、设计） |
| color | string | 标识颜色 |
| order | number | 排序序号 |

### priorities（优先级）— 新增集合（替代固定四阶）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 标识符（如 tian, di, xuan, huang） |
| name | string | 显示名称（天、地、玄、黄） |
| color | string | 颜色值 |
| order | number | 排序序号（越小越高） |

默认优先级保持兼容：
- tian / 天 / #FF3B30 / order:1
- di / 地 / #FF9500 / order:2
- xuan / 玄 / #007AFF / order:3
- huang / 黄 / #34C759 / order:4

## 二、新增 API

### 任务分类管理
| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | /api/categories | 登录用户 | 分类列表 |
| POST | /api/categories | 管理员 | 创建分类 |
| PUT | /api/categories/:id | 管理员 | 编辑分类 |
| DELETE | /api/categories/:id | 管理员 | 删除分类 |

### 优先级管理
| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | /api/priorities | 登录用户 | 优先级列表 |
| POST | /api/priorities | 管理员 | 创建优先级 |
| PUT | /api/priorities/:id | 管理员 | 编辑优先级 |
| DELETE | /api/priorities/:id | 管理员 | 删除优先级 |

### 任务时间更新（甘特图拖拽）
| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| PUT | /api/tasks/:id/dates | 登录用户 | 更新 startDate/endDate |

## 三、甘特图视图设计

### 布局结构
```
┌────────────────────────────────────────────────────────┐
│  工具栏：[卡片|列表|甘特图]  [日|周|月]  [← 今天 →]    │
├──────────────┬─────────────────────────────────────────┤
│  左侧任务列表  │           时间轴区域                     │
│              │  5/12  5/13  5/14  5/15  5/16  5/17     │
│ ▼ 开发       │  ├──────────────────┤                   │
│   任务A      │  │████████████████████│                   │
│   任务B      │       │██████████│                       │
│ ▼ 测试       │                                         │
│   任务C      │            │████████████████│            │
│ ▶ 设计(折叠)  │                                         │
└──────────────┴─────────────────────────────────────────┘
```

### 交互功能

1. **时间粒度切换**：日视图（每格1天）、周视图（每格1周）、月视图（每格1月）
2. **拖拽调整时长**：拖动任务条左/右边缘改变 startDate/endDate
3. **整体移动**：拖动任务条中间区域平移日期（保持时长不变）
4. **点击详情**：点击任务条或左侧任务名打开任务详情面板
5. **分类折叠**：点击分类行的 ▼/▶ 图标折叠/展开该分类下的任务
6. **导航**：左右箭头或滚轮横向滚动时间轴，「今天」按钮回到当前日期

### 技术方案

- 引入轻量甘特图库 **frappe-gantt**（CDN）作为基础渲染引擎
- 在 Vue 层实现分类分组 + 折叠/展开逻辑
- 每个分类组内使用独立 Gantt 实例或统一实例 + 行过滤
- 拖拽事件回调中调用 `PUT /api/tasks/:id/dates` 保存变更

### 任务条样式

- 颜色：使用任务优先级对应的颜色
- 高度：24px，圆角
- 分类行：背景加深，显示分类名和折叠图标
- 今天标记：竖向红色虚线

## 四、系统设置扩展

### 新增设置标签页

- 「分类管理」：CRUD 任务分类（名称、颜色、排序）
- 「优先级管理」：CRUD 优先级（标识、名称、颜色、排序）

### 任务表单更新

- 新增「开始日期」「结束日期」字段
- 新增「任务分类」下拉选择
- 「优先级」改为从 priorities 集合动态读取
- 原「截止日期」更名为「期望时间」

---

## 五、兼容性处理

- 已有任务若无 startDate/endDate，甘特图中不显示或使用 createdAt 作为 startDate、deadline 作为 endDate
- 已有固定优先级（tian/di/xuan/huang）作为 priorities 集合的默认数据
- 未分类任务在甘特图中归入「未分类」组
