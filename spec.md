# 部门任务管理看板系统 — 设计书

基于 Node.js + Express + lowdb 后端和 Vue 3 CDN 单页前端，构建一个苹果风格的部门任务看板管理系统。支持多账号、权限控制、可拖拽泳道看板、悬赏积分排行榜、简单评论、明暗主题切换。

---

## 一、系统架构

```
task/
├── server.js              # Express 服务入口
├── package.json           # 项目依赖
├── db.json                # lowdb 数据文件（自动生成）
└── public/
    └── index.html         # 单文件前端（Vue 3 + CSS）
```

- 后端：Node.js + Express + lowdb，提供 RESTful API
- 前端：单个 index.html，通过 CDN 引入 Vue 3 和 SortableJS
- 通信：fetch API 调用后端接口
- 认证：JWT token，存于 localStorage，请求头 `Authorization: Bearer <token>`
- 部署：局域网内一台电脑运行 `node server.js`，其他人通过 IP 访问

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Vue 3 (CDN) | 无需构建工具，适合单文件应用 |
| 拖拽 | SortableJS (CDN) | 比原生 Drag API 更可靠的跨列拖拽体验 |
| 后端框架 | Express | 轻量 Node.js Web 框架 |
| 数据库 | lowdb | JSON 文件即数据库，零配置 |
| 认证 | jsonwebtoken | 无状态 JWT 认证 |
| 密码 | bcryptjs | 密码哈希，保证基本安全 |
| ID生成 | uuid | 通用唯一标识符 |

## 六、积分规则

- 每个任务创建时设定悬赏分值
- 任务完成后，积分**均分**给所有认领者（总分值 ÷ 认领人数）
- 积分排行榜展示所有成员的累计积分排名

## 七、功能增强（第二轮）

### 7.1 泳道横纵切换

- 看板泳道支持**横向布局**（默认，各泳道列并排显示）和**纵向布局**（各泳道行堆叠，上下排列）
- 顶栏或看板区域提供切换按钮，用户可一键切换
- 切换状态保存在 localStorage，刷新后保持

### 7.2 积分排行位置调整

- 积分排行从看板底部移至**顶部栏区域**（用户信息上方/左侧）
- 始终可见，不随页面滚动消失
- 保持紧凑横向展示样式

### 7.3 系统设置隐藏搜索框

- 当前视图为「系统设置」时，顶栏的"搜索任务"输入框**不显示**
- 仅在看板视图下显示搜索框

### 7.4 用户注册功能

- 登录页面增加**注册入口**，用户可自行注册账号
- 注册字段：用户名、密码、显示名称
- 注册后角色默认为 `member`，分组为空
- 管理员可在「系统设置」中**开启/关闭注册功能**
- 关闭时登录页不显示注册入口，注册 API 返回拒绝

#### 新增 API
| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| POST | /api/auth/register | 公开（受开关控制） | 用户自主注册 |

#### 新增系统设置数据
```json
{
  "settings": {
    "allowRegistration": true
  }
}
```

#### 新增 API（系统设置）
| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | /api/settings | 公开 | 获取公开配置（如是否允许注册） |
| PUT | /api/settings | 管理员 | 更新系统配置 |

### 7.5 任务可见性控制

- **开启时**：
  - 「待认领」泳道（第一个泳道）的任务所有人可见
  - 其他泳道仅显示**当前用户认领的任务**（管理员可见全部）
- **关闭时**：所有用户可看到全部泳道中的所有任务（当前行为）
- 管理员可在「系统设置」中开启/关闭此规则
- 前端通过 API 获取设置状态，根据规则过滤显示

#### 新增系统设置字段
```json
{
  "settings": {
    "taskVisibilityControl": false
  }
}
```

### 7.6 泳道列表模式

- 看板区域支持切换为**列表视图**（与卡片/看板视图并列的第三种展示方式）
- 列表以表格形式展示，每行一个任务，列包含所有属性：
  - 标题、描述、优先级、悬赏分值、截止日期、认领者、创建者、当前泳道、创建时间
- 顶栏或看板区域提供视图切换按钮（卡片视图 / 列表视图）
- 列表模式下仍支持拖拽行来移动泳道（可选），或提供下拉菜单切换泳道

### 7.7 第三轮优化

#### 7.7.1 泳道布局细化

- **工具栏按钮顺序**：横纵切换按钮位于卡片/列表视图切换按钮的**左侧**

- **纵向模式**：
  - 泳道高度自动铺满窗口可用空间（各泳道等分剩余高度，`flex:1`）
  - 泳道宽度自动匹配窗口宽度（`width:100%`），不出现横向滚动条
  - 任务卡片使用固定宽度（260px），泳道内 flex-wrap 横向多列排列
  - 卡片超出泳道高度时，泳道内部滚动

- **横向模式**：
  - 泳道作为列并排展示，各泳道**等分窗口可用宽度**（`flex:1`），不出现横向滚动条
  - 泳道高度铺满页面窗口高度（`align-items:stretch` + `overflow:hidden`）
  - 任务卡片宽度自动适配泳道宽度（`width:100%`），单列纵向排列
  - 卡片超出泳道高度时，泳道内部纵向滚动（`overflow-y:auto`）

#### 7.7.2 菜单图标自定义

- 顶栏导航菜单项支持设置**自定义图标**（emoji）
- 在「系统设置 > 功能开关」中可修改每个菜单项的图标
- 数据结构：
```json
{
  "settings": {
    "menuNames": {
      "board": "看板",
      "settings": "系统设置"
    },
    "menuIcons": {
      "board": "📋",
      "settings": "⚙️"
    }
  }
}
```

#### 7.7.3 搜索栏筛选增强

- 搜索栏区域新增三个下拉筛选：
  - **优先级筛选**：天/地/玄/黄，可选"全部优先级"
  - **认领者筛选**：列出所有出现过的用户，可选"全部认领者"
  - **创建者筛选**：列出所有出现过的用户，可选"全部创建者"
- 筛选条件与搜索关键字、分组筛选同时生效（AND 逻辑）
- 卡片视图和列表视图均受筛选影响

#### 7.7.4 过期任务标识

- 当任务截止时间已过期时，卡片增加醒目的过期标识：
  - 卡片左侧边框强制为红色（`border-left-color: danger`）
  - 卡片背景渐变显示微弱红色提示
  - 标题右侧显示红色「⏰过期」徽章
  - 截止日期文字变为红色加粗
- 列表视图中截止日期列同样以红色高亮显示过期

#### 7.7.5 任务删除权限

- 任务删除操作仅限**管理员**执行
- 后端 `DELETE /api/tasks/:id` 路由使用 `adminOnly` 中间件
- 前端仅对管理员显示删除按钮

## 八、设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 前端方案 | 单HTML + Vue 3 CDN | 降低部署复杂度，无需构建工具 |
| 认证方案 | JWT | 无状态，适合轻量后端 |
| 密码存储 | bcrypt 哈希 | 基本安全保证 |
| 拖拽方案 | SortableJS | 可靠的跨列拖拽 |
| 数据库 | lowdb (JSON文件) | 零配置，适合小团队 |
| 初始管理员 | admin / admin123 | 首次登录提示修改密码 |
