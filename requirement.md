我是一个部门的leader，我想做一个用于部门内任务管理的纯前端web网站。
架构要求如下：
1. 不要后端，只需要纯前端实现，html最好，可以根据评估最优实现方案。
2. 页面风格及布局为苹果风，简约，使用，有高级感。
3. 支持明暗两种主题切换。
4. 尽量让更多的功能和信息在一个页面中展示，但是画面也不要太拥挤影响美观和简洁。

功能需求如下：
1. 支持多账号，能够分组，分配权限。
2. 整体布局参考看板管理的逻辑，可以支持自定义泳道，不同状态的任务可以拖动到对应泳道中。
3. 任务卡片是拖拽方式移动到泳道中的。
4. 任务创建和展示区域要独立，针对任务可以设置悬赏分值，以及期望完成时间。
5. 不同账号登录进来可以进行任务的认领，执行和关闭任务，任务状态根据泳道自动变化。



以上需求如有问题请向我确认，并将确认的结果补充到requirement.md中。

## 需求确认补充（2025-05-15）

### 架构调整
- 前端 + 轻量Node.js后端（使用lowdb，JSON文件作为数据库）
- 部门成员通过局域网访问同一服务，数据共享
- 密码存储在JSON数据库中，实现本地账号验证

### 权限设计
- **管理员**：创建/编辑/删除任务、管理泳道、管理用户账号、验收任务
- **普通成员**：认领任务、执行任务、查看看板

### 任务规则
- 一个任务可由多人协作认领
- 任务设有悬赏分值，完成后积分分配给认领者
- 需要积分排行榜功能

### 默认泳道
1. 待认领
2. 进行中
3. 待验收
4. 已完成

泳道支持管理员自定义增删改。

### 技术栈
- **前端**：单HTML文件 + Vue 3 CDN（或纯原生JS），CSS变量实现明暗主题
- **后端**：Node.js + Express + lowdb
- **拖拽**：HTML5 Drag & Drop API 或 SortableJS
